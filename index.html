##Setting Up Our Dev Environment ###Web Browser **[Google Chrome](https://www.google.com/intl/en-CA/chrome/browser/)** - Choose a modern web browser that supports current web standards and has good development tools. To set Chrome as your default browser, follow [these instructions](https://support.google.com/chrome/answer/95417?hl=en). Another good option is Firefox. Visit [browsehappy.com](http://browsehappy.com) for more info about modern browsers. You can view the source of any web page by right-clicking (or two-finger tap) on any web page! Try visiting http://ladieslearningcode.com and view the source. That's HTML. **Developer tools** take it a step further and allows us to *inspect* any web page. These tools also make it easier to test and debug code. The tools can be accessed in 3 ways: * right-click / two-finger tap on a web page and select **Inspect Element** * select the menu icon in the top right hand corner of the Chrome browser and select **Tools > Developer Tools** * keyboard shortcut: **Ctrl + Shift + I** (PC) / **Cmnd + Option + I** (Mac)

If you are unable to install Sublime Text, there are many free and paid [options](http://en.wikipedia.org/wiki/List_of_HTML_editors). ##Markdown example You can even write in Markdown as long as the `data-markdown` attribute is included. See the [documentation for markdown syntax](http://daringfireball.net/projects/markdown/). Both text and code blocks are supported at this time. Code blocks need to be denoted either by three backticks (`) or by tabbing the content in one indent (at least 4 spaces). Tabbing to create a code block won't work in this slide deck if your content begins tabbed in in your source code. You can specify the language of the codeblock by following the three backticks immediately by the language name. View the source to see this in action. #### Javascript Example ```javascript var myJSVariable = "Syntax Highting for Javascript"; ``` #### CSS Example ```css .css-selector{ highlight : syntax; } ``` #### HTML Example For HTML, set `xml` as the language to get nicer syntax highlighting. Also, the left angled bracket of the HTML tag needs to be escaped. Here's a handy tool for creating escaped result with HTML entities: http://www.htmlescape.net/htmlescape_tool.html ```xml <header> <h1>I'm an H1!</h1> <p>This is a paragraph</p> </header> ``` #### indented example code block that has to guess at the language and likely gets it wrong.
