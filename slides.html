#The Box Model

Padding and Margin

The padding and margin can both be declared using similar syntax of 1 to 4 values. Values can be declared using different measurement units but pixels or percentages are most commonly used. Setting the value to "0" will remove any default space on an HTML element. Any positive value will add space.

  • 4 values means top, right, bottom, left
  • 3 values means top, right/left, bottom
  • 2 values means top/bottom, right/left
  • 1 value means the same amount of padding on all sides

Box Model Review

This diagram represents how all these different properties work together to change the size of an element.

The original element started with a width and height of 400 x 100. After including the padding and border, the size of the box is actually 452 x 152! That's because these properties add space inside the element by pushing the content outward.

Margin takes up space but since it's outside of the element, it doesn't increase the size of the element itself.

Block vs Inline Elements

Right now, our navigation list stacks on top of each other. That’s because they are block elements.
By default, some HTML elements are displayed as block or inline elements.

Block Elements

  • takes up 100% width of its container, no matter how long the actual content is
  • will be the same height as its content
  • always starts on a new line
  • can wrap other block level elements or inline elements
  • can apply sizing related CSS to it (margin, padding, height, width)
I'm a div so that makes a me a block element.

Block vs Inline Elements

Inline Elements

  • will be the height and width of its content
  • always appears “in a line” with other inline level elements
  • can wrap other inline elements but cannot wrap block level elements except &lt;a&gt; tags. This is only valid with an HTML5 doctype.
  • does not render CSS height and width at all, will apply margin and padding but with unexpected results
I'm a span. I'm an inline element.

Class Exercise #5 - Horizontal navigation

In this step, we're focusing on changing the default list styles to a horizontal layout. It will look like this sample below (also found in your examples folder, exercise5-about.html).

First, remove the default list spacing and bullets.

nav ul {
      list-style-type: none; /* removes bullets */
      margin: 0; /* removes default margin */
      padding: 0; /* removes default padding */
    }

Now we’ll want to target each list item to get the elements to line up side by side, in a line.

nav ul li {
    display: inline-block; /* gets the list items to line up side by side */
  }

Add some space around the links. It’s better to put padding on the anchor link rather the the list item because padding adds space on the inside. Putting it on the link with make the click area bigger.

nav a {
  display: block; /* changes the element to block */
  padding: 5px 15px;
  text-transform: uppercase; 
}

Show learners the difference between putting padding in the list item vs anchor. Just demonstrate it with your own code, don't get them to type along so they can just focus on the demo.

Float & Clear

First we have to break it, then put it back together. We need to “reset” the float to get the rest of the page to go back to normal. There are a couple ways to do it.

One of the most confusing CSS concepts is when to use the different techniques.

Let’s look at an example: http://jsfiddle.net/e1nd3q89/

Rule of thumb:
If there is an element that follows the floated elements, apply clear: both to that element. If all the elements need to be floated, apply the overflow:hidden to the parent element to force clear the children.

Class Exercise #7 - Positioning the Header & Nav

Set the entire header to a fixed position so it stays in place at the top of the page on scroll.

header {
    position: fixed;
    top: 0;
  }

Let's check these changes on our About page.

Positioning also takes the element out of the natural stacking order. See how the profile picture “jumped” underneath the header when position: fixed was applied? It’s because it doesn’t “see” that there’s an element before it, so it wants to slide up to the next available slot.

Add some padding on the top of the section element to push the content down below the fixed header. Then try scrolling down the page.

section {
  padding-top: 125px;
}

Update the header css by setting the background to white so the content underneath doesn’t show through when you scroll the page. Also, add a border on the bottom to create a separation of the header from the page content.

header {
  position: fixed;
  background: white;
  border-bottom: 1px solid #eee;
}

Why doesn’t the header go all the way across? When you position an element, it automatically becomes the width of its content. Set the width to 100% so it becomes as wide as the browser. 

header {
  position: fixed;
  background: white;
  border-bottom: 1px solid #eee;
  width: 100%;
}

Next up, position the nav to the top right of the header element. Since the header already has positioning applied to it, the nav will be relative to it.

nav {
  position: absolute;
  right: 20px;
  top: 20px;
}

Class Exercise #9: Background Images (continued)

Thanks to CSS3, we can now change the size of a background image.

background-size is the longhand property. To include it in the shorthand background property, it must be included after background-position, separated with the / character. 

.home {
    background-image: url(../images/background-6.jpg);
    background-repeat: no-repeat;
    background-position: 50%;
    background-size: cover;
  }
.home {
    background: url(../images/background-6.jpg) no-repeat 50% / cover;
  }

If no background position value is being used, background-size won’t work in the shorthand property so in this case, it may be less error prone to add it via the longhand property.

.home {
    background: url(../images/background-6.jpg) no-repeat 50%;
    background-size: cover;
  }

For good measure, since this image is a background image of the body, let's add one more rule to ensure the image covers the whole viewport and not just the height of the page content.

html, body {
  height: 100%;
}
